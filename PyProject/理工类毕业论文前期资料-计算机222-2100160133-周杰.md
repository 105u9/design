| 在全球能源供应日益紧张与我国全面推进"碳达峰、碳中和"双碳战略目标的双重宏观背景下，建筑节能已成为实现绿色可持续发展的关键领域。在我对建筑能耗数据的调研中，我深刻地认识到，在大型公共建筑及企业生产园区中，供暖、通风与空调（HVAC）系统的能源消耗占比高达40%至50%。这不仅给企业带来了沉重的运营成本负担，也对生态环境造成了巨大的碳排放压力。目前，我观察到许多企业仍然采用传统中央空调的固定设定值控制模式，或者依赖于相对简单的比例积分微分（PID）控制逻辑。这种传统的规则控制方法存在一个致命的弱点：它们缺乏足够的灵活性与自适应能力，难以应对外部气象条件的剧烈波动、建筑内部人员流动率的动态变化以及室内热负荷的高度非线性特征。其结果往往导致两种极端情况的发生：要么为了保障人员舒适度而过度制冷或制热，造成令人痛心的能源浪费；要么为了强行降低能耗指标而牺牲了室内环境质量（IAQ），导致员工的舒适度下降，甚至间接影响了企业的生产与工作效率。                                                                                                        |

| 随着物联网（IoT）传感技术、大数据存储以及人工智能（AI）算法的跨越式成熟，我坚信，构建基于数据驱动的智能控制系统已成为解决上述能源浪费与舒适度失衡矛盾的必然趋势。通过在企业现场实时采集环境与设备运行的时序数据，并利用Python语言结合前沿的深度学习算法进行精准预测与动态优化，我们有望在严格保障室内环境舒适度的前提下，实现空调系统的高效、节能与平稳运行。这对于企业降本增效与绿色发展具有极为重要的现实意义。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

| 在深入查阅国内外研究现状后，我发现空调控制系统的发展正经历着一场从"白盒"物理模型向"黑盒/灰盒"数据驱动模型的深刻范式转变。^\[5\]^传统的模型预测控制（MPC）极度依赖于诸如EnergyPlus或TRNSYS等底层物理引擎构建的白盒模型。然而，白盒模型的实施成本极其高昂，参数校准过程繁琐，且在实际运行中容易因建筑物理特性的老化而产生无法弥补的偏差。近年来，学术界与工业界已经将目光全面转向了基于人工智能的数据驱动预测模型。相关的权威研究表明，通过在线学习增强的数据驱动模型预测控制策略，能够同时响应室内污染物浓度与热舒适度需求，并最小化HVAC系统的能耗。例如，一项发表于2025年的最新研究证实，采用在线学习优化器的MPC策略，相较于传统的基线反馈控制，不仅能使二氧化碳峰值浓度显著降低23.2%，还能使总能耗大幅降低10.2%。^\[1\]^此外，针对工业现场往往存在数据缺失的现实工程问题，有学者提出了一种基于信息完整度的自适应切换控制模型，该模型通过迁移模仿学习，在低数据完整度条件下比纯模型优化提升了11.5%的运行效益，进一步降低了5.5%的运营成本与5.0%的能耗。^\[2\]^ |

| 在发展趋势方面，我注意到深度学习算法，尤其是长短期记忆网络（LSTM）及其变体门控循环单元（GRU），在建筑负荷预测领域展现出了无可比拟的优势。最新的文献指出，结合空间-时间注意力机制（Spatiotemporal Attention）的增强型LSTM网络能够有效提取深层时间步特征，通过自适应调整各个变量（如室外温度、太阳辐射、室内人员密度）的权重，极大地提升了建筑能耗与热负荷预测的精度。^\[3\]^此外，在物联网系统集成的智能建筑中，采用时间序列预测与强化学习相结合的数据驱动预测控制，已成为当前最前沿的研究热点。^\[8\]\[9\]^                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

| 2．课题的研究目标、内容和拟解决的关键问题                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

| 基于上述背景与意义，我的研究目标是针对某企业的实际空调运行场景，设计并实现一套具备高鲁棒性、高精度的基于数据驱动的智能控制系统。具体细化为以下四个维度的目标：                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 第一，节能优化目标。我期望通过引入先进的机器学习算法与智能控制策略，使目标企业的空调系统整体能耗较传统固定温度控制模式降低10%以上，切实为企业带来经济效益。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 第二，舒适度保障目标。系统必须能够动态感知室内环境，并将室内热舒适度严格维持在预期的平均热感觉评价（PMV）或不适指数（DI）的舒适区间内，实现"以人为本"的环境控制，绝不能以牺牲员工健康为代价换取节能。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 第三，系统智能化闭环目标。我致力于实现从底层传感器数据采集、算法模型预测到终端控制指令下发的完整自动化闭环。通过算法赋能，减少人工日常干预，使系统具备应对动态天气与人流变化的自适应学习能力。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 第四，原型验证与可视化目标。我将发挥计算机专业的优势，开发一个具备直观数据可视化监控界面的Web软件原型系统，并通过模拟数据集或仿真工具验证核心AI算法的有效性与工程可行性。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

| 二是长序列预测中的误差累积与动态适应性问题。建筑热环境是一个动态时变系统，多步预测容易产生误差累积。我需要研究如何优化LSTM的网络结构，确保其能够精准捕获长距离的时间依赖关系。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |

**2.文献综述**

**2.1推荐系统：发展历程与应用**

推荐系统已成为各个领域不可或缺的一部分，通过预测偏好和推荐相关商品，实现个性化的用户体验^\[13\]^。推荐系统最初在亚马逊和Netflix等电子商务和娱乐平台中占据重要地位，如今已扩展到智能建筑管理、医疗保健和城市规划等各个领域^\[14\]^。

**2.1.1.推荐系统的类型**

推荐系统主要分为三种类型：协同过滤、基于内容的过滤和混合方法。

协同过滤（CF）：依赖于用户与物品之间的交互来识别用户或物品之间的模式和相似性。CF可以进一步分为基于用户的方法和基于物品的方法^\[15\]^。然而，CF存在可扩展性问题和冷启动问题，即新用户或物品缺乏足够的数据来进行准确的推荐。

‧基于内容的过滤（CBF）：利用商品属性和用户画像，向用户推荐与其之前互动过的商品相似的商品^\[16\]^。虽然CBF缓解了CF的一些局限性，但它需要详细的商品元数据，并且可能导致过度专业化，从而降低推荐的多样性。

‧混合推荐系统：结合协同过滤（CF）和基于条件的滤波（CBF）算法，以发挥两种方法的优势并弥补它们的不足^\[17\]^。混合方法提高了推荐的准确性和鲁棒性，使其适用于智能建筑中暖通空调优化等复杂应用。

**2.1.2.高级推荐系统技术**

推荐系统的最新进展主要集中在集成机器学习算法、深度学习模型等方面。

以及基于图的方法来提高预测准确性和可扩展性^\[18\]^。

‧基于深度学习的推荐系统：利用神经⽹络捕捉复杂的用户‑物品交互和潜在特征。

自编码器、卷积神经⽹络(CNN)和循环神经⽹络(RNN)等技术已证明在推荐性能方面有显著改进^\[19\]^。

‧基于图的推荐系统：利用图结构对用户、物品和其他实体之间的关系进行建模。图神经⽹络（GNN）和图嵌入技术在捕捉推荐系统中固有的结构和语义信息方面特别有效^\[20\]^。

**2.2推荐系统中的集成学习**

集成学习结合多个模型，通过减轻单个模型的偏差和方差来提高整体预测性能^\[21\]^。集成方法已被用于提高推荐系统的准确性、鲁棒性和可扩展性。

**2.2.1.集成方法的类型**

‧Bagging（自助聚合）：涉及在训练数据的不同子集上训练同一基础模型的多个实例，并通过投票或平均来聚合它们的预测结果^\[22\]^。Bagging可以降低方差并防止过拟合，使其适用于决策树等高方差模型。

‧提升算法：通过重点关注先前被错误分类的实例来顺序训练模型，从而减少偏差并提高模型准确率^\[23\]^。诸如AdaBoost和GBM之类的算法因其卓越的性能而被广泛应用于推荐系统中。

‧堆叠（堆叠泛化）：通过训练一个元模型来学习多个基础模型预测结果的最佳组合，从而将多个基础模型结合起来^\[24\]^。堆叠允许集成不同的模型，从而增强整体预测能力。

**2.2.2.HVAC推荐系统中的集成学习**

在暖通空调（HVAC）优化中，集成学习已被用于整合各种数据源和建模方法，从而增强系统预测和适应动态环境及用户条件的能力。例如，Goya等人（2018）证明，通过结合决策树、支持向量机和神经⽹络，集成模型在预测最佳暖通空调设置方面优于单一分类器。然而，现有的集成方法通常优先考虑能源优化，而忽略了室内空气质量（IAQ）的维护，这可能导致居住者的健康和舒适度受到影响^\[25\]^。

**2.3图嵌入技术**

图嵌入是指将图结构数据转换为低维向量表示，同时保留图的结构属性和语义信息^\[26\]^。这些嵌入有助于将机器学习算法应用于图数据，从而实现节点分类、链接预测和聚类等任务。

**2.3.1.图嵌入的基础知识**

图嵌入技术旨在将节点、边和子结构编码到连续向量空间中。其主要目标包括：

保留邻域结构，捕捉更高阶的邻近性，并保持语义相似性^\[27\]^。

‧基于随机游走的方法：诸如DeepWalk和Node2Vec之类的技术在图上执行随机游走以生成序列。

节点序列，然后将其输入语言模型以学习嵌入^\[27,28\]^。

下：

1.软件架构的稳健性：如图6所示，该架构框架展现出结构良好且模块化的设计，有助于高效的数据采集、处理和用户交互。集成⾝份验证和监控等辅助服务，进一步增强了系

3.传感器数据相关性：如图1所示，对传感器数据的分析识别出⼏个高度相关的传感器对。值得注意的是，温度和二氧化碳浓度（传感器1和传感

4.居住者满意度：图9显示，居住者的满意度受舒适度、照明和空气质量等环境因素的显著影响。舒适度和空气质量等类别的平均评分较高，而噪音和隐私等方面则被认为是

需要改进的领域。

**5.2结果解读**

**5.2.1.软件架构的鲁棒性和可扩展性**

该软件架构能够在负载不断增加的情况下保持稳定的性能，这证明了其设计的稳健性。

这种模块化分层结构涵盖数据摄取、处理、业务逻辑和API⽹关确保每个组件都可以根据需求独立扩展。例如，数据处理层可以水平扩展以处理更大的数据量。

传入数据而不影响前端/用户界面的响应速度。

然而，尽管可扩展性测试表明，在1000个并发用户的情况下，该架构的性能令人满意，但必须认识到，实际应用场景可能会引入更多复杂因素，例如⽹络延迟、意

外的数据峰值或硬件限制。未来的研究应纳入更多样化、更贴近实际的负载模式，以进一步验证该架构的可扩展性。

**5.2.2传感器数据相关性及其意义**

传感器对之间显著的相关性分析，为深入了解建筑内环境因素的相互关联性提供了宝贵的见解。例如，温度和二氧化碳浓度（传感器1和传感器10）之间存在强

相关性，这表明随着人员密度增加，二氧化碳排放量上升，温度也可能随之升高，这可能是由于人体发热和设备使用增加所致。这种相互依存关系凸显了集成控制系

统的重要性，此类系统能够同时管理多个环境参数，从而维持最佳环境条件。

此外，湿度与人员占用率（传感器2和传感器7）之间的相关性表明，较高的人员占用率可能导致湿度升高，进而影响人员舒适度和空气质量。了解这些关系有助于

开发预测模型和自动响应机制，从而维持平衡舒适的环境。

此外，对Node2Vec、GraphSAGE和GAT等不同图嵌入技术的比较表明，它们在捕捉传感器间相互依赖关系方面具有不同的有效性。GAT的平均余弦相似度最

高，表明其在捕捉细微关系方面具有卓越的能力，而Node2Vec则因其计算效率高而著称，使其适用于实时应用。这凸显了根据暖通空调优化任务的具体需求选择合

适的嵌入方法的重要性。

**5.2.3.居住者满意度和环境因素**

住户满意度指标显示，舒适度、照明和空气质量对用户在建筑内的满意度至关重要。这些类别的高满意度表明，当前的环境控制措施有效地解决了这些方面的问

题。

反之，噪音和隐私等方面的满意度较低，则凸显了有针对性改进的机会。

舒适度和空气质量满意度与其对应的传感器读数之间存在正相关性，这验证了系统在管理这些参数方面的有效性。然而，与噪音和隐私相关的不满可能源于环

境控制以外的因素，例如建筑设计、声学效果或个人工作空间布局。解决这些问题可能需要多学科协作，包括建筑重新设计或引入降噪技术。

此外，整合用户反馈机制可以提供关于满意度的实时数据，从而实现自适应

调整环境控制措施，并更积极主动地解决不满之处。

**5.2.4.技术性能与用户满意度的整合**

稳健的软件架构、可扩展的系统性能以及高度的用户满意度三者相融合，体现了技术基础设施与用户体验的成功结合。高效的数据管理和响应迅速的控制系统

直接有助于维持提升用户舒适度和满意度的环境条件。

然而，该研究也强调需要采取整体方法，将技术因素和以人为中心的因素都考虑在内。

虽然该系统能够有效管理环境参数，但噪音和隐私等显著影响用户满意度的因素，可能需要除传感器数据控制之外的补充策略。整合用户反馈机制和能够学习并响

应用户偏好的自适应系统，可以进一步提升用户满意度。

此外，集成学习框架（包括决策树、随机森林、XGBoost和神经⽹络）在提升预测精度和系统响应速度方面发挥了关键作用。每个模型都对集成模型做出了独特

的贡献：神经⽹络提供深度模式识别，随机森林增强了抗过拟合的鲁棒性，而XGBoost则优化了预测性能。这种协同组合确保了推荐系统能够根据动态环境和人员

占用情况，提供精准可靠的暖通空调(HVAC)设置。

3.以用户为中心的设计：将用户满意度指标纳入系统评估，可确保技术解决方案的有效性。

符合用户需求，营造更舒适、更高效的工作环境。

4.持续监测和适应：实施综合监测工具可以实时评估系统性能和环境条件，从而进行及时干预和调整。

求的解决方案的开发，从而增强了系统在实际环境中的相关性和适用性。

**5.4研究的局限性**

虽然这项研究提供了宝贵的见解，但仍需指出其存在一些局限性：

1.合成数据使用：依赖合成数据来获取传感器读数和用户满意度信息可能无法全面反映真实场景的复杂性和多变性。未来的研究应纳入实际传感器数据和真实用户反馈，以验

证研究结果。

2.环境参数范围有限：该研究仅关注了部分传感器和满意度类别。

纳入更广泛的环境和人体工程学因素，可以更全面地了解乘员满意度的驱动因素。

3.静态相关性分析：皮尔逊相关性分析假设数据之间存在线性关系，但并未考虑传感器之间可能存在的非线性相互作用。更高级的统计方法或机器学习技术或许能够揭示更复

杂的模式。

4.单一建筑环境：研究结果置于特定的建筑环境中进行分析。建筑设计和用途的变化都会影响研究结果。

模式和居住者人口统计特征可能会影响结果的普遍性。

**5.5对未来研究的建议**

为了克服上述局限性并在此研究成果的基础上，提出以下建议：

1.整合真实世界数据：利用实际传感器读数并收集真实的居住者反馈，以提高有效性和可靠性。

结果的适用性。

2.扩展环境和人体工程学指标：增加其他传感器和满意度类别，例如照明强度、声音水平、工作空间人体工程学和热舒适度，以更全面地了解居住者的满意度。

3.运用高级分析技术：探索非线性相关性测量、主成分分析或机器学习模型，以识别环境参数之间的复杂关系及其对用户满意度的影响。

4.纵向研究：开展长期研究，观察系统性能和居住者满意度的变化。

随着时间的推移而演变，尤其是在建筑用途或外部因素发生变化时。

5.跨建筑比较：将研究范围扩大到多个设计和使用模式各异的建筑，以评估⋯⋯

软件架构的通用性及其对不同环境下用户满意度的影响。

6.自适应控制系统的集成：开发和评估自适应控制机制，根据实时传感器数据和乘员偏好动态调整环境参数，进一步提高用户满意度和系统效率。

1.建筑师和工程师：可以利用强大且可扩展的软件架构框架来设计能够有效管理环境参数的智能建筑系统。

3.住户健康计划：通过识别显著影响满意度的环境因素，可以制定相应的计划。

针对特定问题进行定制，例如降低噪音水平或增强隐私。

4.技术提供商：深入了解传感器之间的关键关联性可以指导开发更集成化的系统。

响应式传感器⽹络，增强智能建筑解决方案的整体效能。

行业标准和消费者需求促进了智能建筑技术的采用和成功。

6.结论

本研究强调了稳健且可扩展的软件架构对于有效管理智能建筑中的环境和人员占用数据至关重要。我们的研究结果表明，所提出的架构不仅能在显著的负载变化下保持高

性能，还能巧妙地处理关键环境参数之间复杂的关联。研究发现，温度和二氧化碳浓度、湿度和人员占用情况以及空气质量等因素之间存在着密切的联系。

质量和二氧化碳水平凸显了建筑环境的相互关联性以及集成控制系统的必要性。

此外，居住者满意度分析表明，舒适度、照明和空气质量是影响用户幸福感的关键因素，这证实了系统性能与以用户为中心的结果之间的一致性。这些类别的高满意度表明，当前的环

境控制措施有效地解决了这些方面的问题。相反，噪声管理和隐私等领域则显示出进一步改进的空间，这表明技术解决方案必须与建筑和设计方面的考量相结合，才能全面提升居住者的

满意度。

尽管该研究具有诸多优势，但由于其依赖于合成数据和受控环境，因此研究结果的普遍适用性受到限制。

未来的研究应侧重于在不同建筑类型和环境中进行实际应用，以验证和拓展这些发现。此外，探索先进的分析方法和机器学习技术可以揭示传感器数据中更复杂的模式，从而进一步提升

系统的智能性和响应能力。

总之，将可扩展的软件架构与数据驱动的环境控制和用户满意度指标相结合，为打造智能建筑提供了一条充满希望的途径。这些智能建筑不仅高效运行，还能为居住者营造健康舒适

的环境。这种整体方法为未来智能建筑技术的进步奠定了基础，强调了技术稳健性和以人为本的设计之间的协同作用。通过兼顾技术和人为因素，智能建筑可以实现可持续的智能气候控

制，最终提升居民的生活质量。
